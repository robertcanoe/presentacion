¿Qué es React en 2026?

Imagina que quieres construir Instagram o Netflix desde cero. React es tu caja de herramientas: componentes reutilizables que se actualizan automáticamente cuando cambian los datos.

  • Librería UI — enfocada 100% en la interfaz de usuario
  • Basado en componentes — construyes la UI como piezas de LEGO
  • JSX — HTML dentro de JavaScript: más poder y flexibilidad
  • Hooks modernos — gestión de estado y efectos con funciones puras

React: La Evolución

React ha evolucionado radicalmente. Lo que antes era complejo ahora es simple y potente.

❌ React Antiguo (v15-v16)

Clases obligatorias, this.setState, métodos lifecycle complicados, HOCs anidados

✅ React 19 (hoy)

Hooks puros, componentes funcionales, Server Components, Compiler automático, Actions

Componentes en React

La base de tu aplicación

Un componente = función JavaScript que devuelve JSX (HTML + JS).
Como bloques de construcción: autónomos, reutilizables y combinables.

Antes vs Ahora: Componentes

Antes usábamos clases complicadas. Ahora todo son funciones simples.

❌ React Antiguo

class User extends Component {

render() {

return <h2>{this.props.name}</h2>

}

}

✅ React 19

function User({ name }) {

return <h2>{name}</h2>

}

Tu Primer Componente

💡 Los componentes son funciones que devuelven JSX. Las props son como parámetros de función.

function Saludo({ nombre }) {

return (

<div>

<h1>¡Hola {nombre}!</h1>

</div>

)

}

// Usar el componente

<Saludo nombre="React 19" />

JSX: HTML + JavaScript

💡 JSX permite escribir estructura HTML dentro de JavaScript. Las llaves {} evalúan expresiones JS.

function Producto() {

const titulo = 'MacBook Pro';

const precio = 1999;

const disponible = true;

return (

<div>

<h2>{titulo}</h2>

<p>Precio: {precio}€</p>

<p>Stock: {disponible ? 'Sí' : 'No'}</p>

</div>

)

}

Props: pasar datos entre componentes

💡 Las props son argumentos que pasas a un componente. Fluyen de padre a hijo (unidireccional).

function Tarjeta({ imagen, titulo, precio }) {

return (

<div>

<img src={imagen} alt={titulo} />

<h3>{titulo}</h3>

<p>{precio}€</p>

</div>

)

}

// Uso

<Tarjeta imagen="img.jpg" titulo="Laptop" precio={999} />

React Hooks

Superpoderes para componentes funcionales

Los Hooks son funciones especiales que te permiten "engancharte" a características de React
(estado, ciclo de vida, contexto) desde componentes funcionales.

useState: gestionar estado local

💡 useState devuelve [valor, función]. Cada vez que llamas a la función, React re-renderiza el componente con el nuevo valor.

import { useState } from 'react';

function Contador() {

const [contador, setContador] = useState(0);

return (

<div>

<p>Contador: {contador}</p>

<button onClick={() => setContador(contador + 1)}>

+1

</button>

</div>

)

}

useEffect: efectos secundarios

💡 useEffect ejecuta código después del render. Perfecto para fetch, suscripciones, manipular DOM.

import { useState, useEffect } from 'react';

function Titulo() {

const [contador, setContador] = useState(0);

useEffect(() => {

// Se ejecuta cada vez que cambia contador

document.title = `Clicks: ${contador}`;

}, [contador]);

return (

<button onClick={() => setContador(contador + 1)}>

Haz clic

</button>

)

}

useEffect: las 3 variantes

💡 El array de dependencias controla cuándo se ejecuta el efecto. Olvidarlo es el error más común en React.

// ❌ Sin array → se ejecuta en CADA render

useEffect(() => { console.log('render!') });

// ✅ [] → solo al montar el componente (mount)

useEffect(() => { cargarDatos() }, []);

// ✅ [dep] → cada vez que cambia "query"

useEffect(() => { buscar(query) }, [query]);

// ♻️ Cleanup → se ejecuta al desmontar el componente

useEffect(() => {

const timer = setInterval(() => { ... }, 1000);

return () => clearInterval(timer); // limpia al desmontar

}, []);

useRef: referencias sin re-render

💡 useRef tiene 2 usos: acceder directamente a un elemento del DOM, y guardar un valor que persiste entre renders sin causar re-render (al contrario que useState).

import { useRef } from 'react';

// 1️⃣ Acceder a un nodo del DOM directamente

function Busqueda() {

const inputRef = useRef(null);

const enfocar = () => inputRef.current.focus();

return (

<>

<input ref={inputRef} placeholder="Buscar..." />

<button onClick={enfocar}>Enfocar</button>

</>

)

}

// 2️⃣ Guardar un valor sin causar re-render

const renderCount = useRef(0);

renderCount.current++; // cambia el valor pero NO re-renderiza

Listas y Keys

💡 Usa .map() para renderizar listas. La key debe ser única y estable (identifica cada elemento).

function ListaTareas() {

const tareas = [

{ id: 1, texto: 'Aprender React' },

{ id: 2, texto: 'Crear proyecto' }

];

return (

<ul>

{tareas.map(tarea => (

<li key={tarea.id}>{tarea.texto}</li>

))}

</ul>

)

}

Key: ¿por qué NO usar el índice?

💡 React usa la key para rastrear qué elemento es qué. Si usas el índice como key y borras o reordenas items, React reutiliza el DOM incorrecto → bugs de UI.

❌ MAL — key con índice

// Si borras la tarea 0, la tarea 1

// pasa a ser key=0 → React la reutiliza mal

tareas.map((t, i) => (

<li key={i}>

{t.texto}

</li>

))

✅ BIEN — key con id único

// La key es estable aunque

// cambien el orden o se borren

tareas.map((t) => (

<li key={t.id}>

{t.texto}

</li>

))

👉 Genera IDs con Date.now(), crypto.randomUUID() o un simple contador incremental.

Actualizar arrays y objetos en estado

💡 Nunca mutéis el estado directamente. Siempre cread una copia nueva — React no detecta cambios si mutáis el original.

// ❌ MAL: mutar directamente → React NO re-renderiza

state.push(nueva); setState(state);

// ✅ AÑADIR — spread

setTareas([...tareas, nuevaTarea]);

// ✅ BORRAR — filter

setTareas(tareas.filter(t => t.id !== id));

// ✅ MODIFICAR — map + spread del objeto

setTareas(tareas.map(t =>

t.id === id ? { ...t, completada: true } : t

));

// ✅ ACTUALIZAR OBJETO — spread

setUsuario({ ...usuario, nombre: 'Ana' });

Renderizado Condicional

💡 Usa operadores JavaScript (&&, ? :) para mostrar u ocultar elementos según condiciones.

function Mensaje({ usuario }) {

return (

<div>

{/* Con && */}

{usuario && <p>Hola {usuario}</p>}

{/* Con ternario */}

{usuario ? (

<p>Bienvenido {usuario}</p>

) : (

<p>Inicia sesión</p>

)}

</div>

)

}

Formularios controlados

💡 Un input controlado mantiene su valor en el estado de React. El estado es la única fuente de verdad — el input sólo muestra lo que React dice.

function Formulario() {

const [nombre, setNombre] = useState('');

const handleSubmit = (e) => {

e.preventDefault(); // evita recarga de página

console.log('Enviado:', nombre);

};

return (

<form onSubmit={handleSubmit}>

<input

value={nombre}

onChange={(e) => setNombre(e.target.value)}

placeholder="Tu nombre"

/>

<button type="submit">Enviar</button>

</form>

)

}

Lifting state up: hijo → padre

💡 En React el estado fluye hacia abajo (props). Para que un hijo notifique al padre, el padre pasa una función callback como prop.

// Padre — tiene el estado y pasa la función

function Padre() {

const [valor, setValor] = useState('');

return <Hijo onChange={setValor} />;

}

// Hijo — llama al callback del padre

function Hijo({ onChange }) {

return (

<input

onChange={(e) => onChange(e.target.value)}

/>

)

}

Patrón clave: estado en el ancestro común más cercano de todos los componentes que lo necesitan.

Volveremos con Router y nuevos Hooks de React 19

React Router

Navegar entre páginas

React Router permite crear aplicaciones multi-página (SPA).
Cada URL muestra un componente diferente sin recargar.

Instalar React Router

# React Router v7 (la que usamos)

npm install react-router-dom@7

Configurar Rutas

💡 createBrowserRouter define las rutas en un array (main.tsx). App usa <Outlet /> donde se renderizan los hijos.

// main.tsx

import { createBrowserRouter, RouterProvider } from 'react-router-dom';

const router = createBrowserRouter([

{ path: '/', element: <App />, // ← App tiene <Outlet />

children: [

{ index: true, element: <Home /> },

{ path: 'productos', element: <Productos /> },

{ path: 'producto/:id', element: <Detalle /> },

],

},

]);

// En main.tsx — sustituye <App /> por RouterProvider

<RouterProvider router={router} />

// En App.tsx — Outlet marca dónde se renderiza el hijo activo

import { Outlet } from 'react-router-dom';

function App() { return ( <div><nav>...</nav><Outlet /></div> ) }

Link y NavLink: navegar sin recargar

💡 Usa Link para navegar sin recargar. Usa NavLink cuando necesites saber si el enlace está activo (para estilarlo).

import { Link, NavLink } from 'react-router-dom';

function Nav() {

return (

<nav>

{/* Link — navegación normal */}

<Link to="">Home</Link>

{/* NavLink — añade clase 'active' al enlace actual */}

<NavLink

to="/productos"

className={({ isActive }) => isActive ? 'active' : ''}

>

Productos

</NavLink>

</nav>

)

}

useParams: capturar parámetros

import { useParams } from 'react-router-dom';

function Detalle() {

const { id } = useParams();

return <h2>Producto #{id}</h2>;

}

Fetch API

Obtener datos de APIs

Fetch es la función nativa del navegador para hacer peticiones HTTP.
Combínala con useEffect para cargar datos al montar el componente.

Fetch con useEffect

💡 Llama a fetch dentro de useEffect con array vacío [] para ejecutar solo al montar.

function Usuarios() {

const [usuarios, setUsuarios] = useState([]);

useEffect(() => {

fetch('https://jsonplaceholder.typicode.com/users')

.then(res => res.json())

.then(data => setUsuarios(data))

.catch(err => console.error(err));

}, []);

return (

<ul>

{usuarios.map(u => (

<li key={u.id}>{u.name}</li>

))}

</ul>

)

}

Nuevos Hooks React 19

Actions, useOptimistic, useFormStatus

React 19 introduce nuevos hooks para formularios,
actualizaciones optimistas y mejor gestión de estado asíncrono.

useActionState: gestionar acciones asíncronas

💡 Nuevo hook que gestiona el estado de acciones asíncronas (loading, error, data) automáticamente.

import { useActionState } from 'react';

async function guardarDatos(data) {

const res = await fetch('/api/save', {

method: 'POST',

body: JSON.stringify(data)

});

return await res.json();

}

function Form() {

const [state, action] = useActionState(guardarDatos);

return <form action={action}>...</form>;

}

useFormStatus: estado del formulario

💡 Hook que te da el estado del formulario padre (pending, data). Perfecto para botones de submit.

import { useFormStatus } from 'react-dom';

function SubmitButton() {

const { pending } = useFormStatus();

return (

<button

type="submit"

disabled={pending}

>

{pending ? 'Enviando...' : 'Enviar'}

</button>

)

}

useOptimistic: UI optimista

💡 Actualiza la UI inmediatamente mientras se procesa en background. Si falla, revierte automáticamente.

import { useOptimistic } from 'react';

function Likes({ count }) {

const [optimisticCount, setOptimistic] =

useOptimistic(count);

async function handleLike() {

setOptimistic(optimisticCount + 1);

await fetch('/api/like', { method: 'POST' });

}

return (

<button onClick={handleLike}>

❤️ {optimisticCount}

</button>

)

}

use: leer recursos en render

💡 Nuevo hook que permite leer Promises y Context directamente en el render.

import { use } from 'react';

function Usuario({ promesa }) {

// Suspende hasta que se resuelva

const user = use(promesa);

return <h2>{user.name}</h2>;

}

React Server Components (RSC)

Componentes que se ejecutan en el servidor. Solo envían HTML al cliente — JavaScript cero.

  • Mejor rendimiento — menos JS en el navegador
  • Acceso directo a datos — fetch en servidor sin API
  • SEO mejorado — HTML completo desde el servidor
  • Disponible en Next.js 14+

React Compiler (Experimental)

Compilador que optimiza automáticamente tu código React sin necesidad de useMemo o useCallback.

  • Memoización automática de componentes
  • Optimiza re-renders innecesarios
  • Reduce boilerplate de optimización
  • Experimental en React 19

Custom Hooks: reutilizar lógica

💡 Crea tus propios hooks para extraer y reutilizar lógica. Deben empezar con "use".

function useFetch(url) {

const [data, setData] = useState(null);

const [loading, setLoading] = useState(true);

const [error, setError] = useState(null);

useEffect(() => {

async function cargar() {

try {

const res = await fetch(url);

setData(await res.json());

} catch (err) {

setError(err.message);

} finally {

setLoading(false);

}

}

cargar();

}, [url]);

return { data, loading, error };

}
